    ИСПРАВЛЕНО
	Нерегулярная ошибка с переносами сальдо при добавлении контрагента в список 
	"Сальдо с контрагентами"
	При выборе значений из справочников больше не отключается цифровой блок клавиатуры
	(NumLock)! 	Программная эмуляции нажатия клавиш в коде VBA (SendKeys), явившаяся 
	причиной такого поведения программы, заменена другими конструкциями. Проблема 
	присутствовала с самых первых версий ПО (2001 год) и, наконец, решена (кроме блока
	Товарно-денежных отчётов).  
	Список ТТН на сторону: результат поиска сортируется не по возрастанию даты
	Список счетов на оплату (Счёт/Договор): результат поиска отображается без копеек
	Список приходных ордеров: результат поиска отображается без копеек
	Список Сальдо с контрагентами: результат поиска отображается без копеек
	Отчёт Акт сверки: расширено поле "Дата печати отчёта"
	Отчёт Расчёты с поставщиками: расширено поле "Дата печати отчёта"
	Отчёт Анализ приходных накладных (Ctrl+F2): результат отображается без копеек
	Подсказка в строке состояния формы Оперативный учёт (добавление/удаление записей здесь недоступно, это делается в форме Сальдо с контрагентами)
	Мелкие некритичные ошибки
